# コンプライアンス運用手順書 (Compliance Runbook)

## 1. 概要
本手順書は、`mars_lite` における予測モデルのデプロイおよび運用において、規制遵守（コンプライアンス）および監査要件を満たすために必要な証跡（エビデンス）の管理、管轄区域チェックリスト、およびデータの保存ポリシーについて定義します。

## 2. 必須の証跡 (Required Evidence)
モデルを次のデプロイステージへ昇格（プロモーション）させるたびに、以下の証跡を永続的に保存し、監査可能な状態にする必要があります。

- **モデルレジストリ情報**: 登録されたモデルバージョンおよびアクティブバージョン。
- **学習マニフェスト**: Gitコミットハッシュ、学習データのハッシュ、ハイパーパラメータ、および乱数シード（seed）。
- **ドリフト監視レポート**: プロモーション評価ウィンドウにおけるドリフト検出結果のレポート。
- **デプロイゲート証跡**: Shadow、Canary、Productionの各ステージでのデプロイゲート評価結果（判定ログ）。
- **人間による承認チケット**: Productionステージ昇格時に必要となる、承認されたチケット番号（例：`PROD-123`）。

## 3. 証跡の保管と整合性検証手順 (Evidence Storage and Integrity Verification)

### 3.1 証跡の保管場所
すべての証跡は、各プロモーションステージおよびモデルバージョンごとに以下のいずれかの場所に保管されます。

- **クラウドストレージ（推奨）**: `s3://trade-rl-audit-trail/promotions/<stage>/<model_version>/`
- **ローカル環境（フォールバック）**: `output/audit/promotions/<stage>/<model_version>/`

### 3.2 証跡ファイルの命名規則
各ディレクトリには以下のファイルを配置し、対応する命名規則を厳守してください。

- **学習マニフェスト**: `manifest.json`
- **ドリフト監視レポート**: `drift_report.json`
- **デプロイゲート評価結果**: `gate_results.json`
- **人間による承認チケット**: `approval_ticket.txt`

### 3.3 SHA-256 ハッシュ値による整合性検証手順
改ざん検知および整合性確認のため、すべての証跡ファイルは保存時に SHA-256 ハッシュ値を算出し、マニフェストファイルまたは監査ログに記録します。
検証担当者は以下の手順を実行して整合性を検証してください。

1. **ローカルまたはS3から対象ファイルのハッシュ値を計算する**:
   Windows PowerShellの場合：
   ```powershell
   Get-FileHash -Path "output/audit/promotions/<stage>/<model_version>/manifest.json" -Algorithm SHA256
   ```
   Linux/macOSの場合：
   ```bash
   sha256sum output/audit/promotions/<stage>/<model_version>/manifest.json
   ```

2. **基準値との比較**:
   計算されたハッシュ値と、プロモーション承認時に台帳（あるいはリリース通知）に記録された SHA-256 ハッシュ値を突き合わせます。ハッシュ値が一致しない場合、証跡の改ざんまたは破損とみなし、即座にインシデント（SEV1/SEV2）として起票します。

## 4. 管轄チェックリスト (Jurisdiction Checklist)
各監査領域において、以下の証跡が適切に記録されていることを確認します。

| 領域 (Area) | 必要な証跡 (Evidence) |
| --- | --- |
| **データの出所 (Data provenance)** | 取得元ソース、対象時間範囲、データハッシュ値 |
| **モデルの再現性 (Model reproducibility)** | 学習マニフェスト、およびモデルレジストリに登録されたバージョン情報 |
| **リスク管理 (Risk control)** | 事前取引（Pre-trade）リスクチェックおよびデプロイゲートの評価レポート |
| **監査可能性 (Auditability)** | モデル意思決定ログ（`model_decision_log.md`）への記録エントリー |

## 5. 証跡の保存期間 (Retention)
デプロイプロモーションの証跡、インシデント記録、およびモデル意思決定ログは、管轄する法域または取引所のポリシーでより厳格な基準が適用されない限り、**最低7年間**保存しなければなりません。
